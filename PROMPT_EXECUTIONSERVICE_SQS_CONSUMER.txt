ESTRUTURA ESPERADA (para referência)
====================================

Com base em como BillingService implementou, esperamos que ExecutionService tenha:

```csharp
// 1. HostedService que inicia na aplicação
public class SqsEventConsumerHostedService : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            var messages = await _sqs.ReceiveMessageAsync(
                queueUrl: Environment.GetEnvironmentVariable("AWS_SQS_QUEUE_BILLING"),
                maxMessages: 10,
                waitTimeSeconds: 5
            );
            
            foreach (var message in messages)
            {
                // Parse e route para handler correto
                var eventType = ExtractEventType(message);
                
                if (eventType == "PaymentConfirmed")
                    await _paymentConfirmedHandler.HandleAsync(message);
                
                // Delete da fila
                await _sqs.DeleteMessageAsync(message);
            }
        }
    }
}

// 2. Handler específico
public class PaymentConfirmedHandler
{
    public async Task HandleAsync(Message message)
    {
        var payment = JsonSerializer.Deserialize<PaymentConfirmed>(message.Body);
        
        // Cria job de execução
        var job = new ExecutionJob { OsId = payment.OsId, Status = "Queued" };
        await _db.ExecutionJobs.AddAsync(job);
        await _db.SaveChangesAsync();
        
        // Executa ações (diagnósticos, reparação, etc)
        var result = await ExecuteAsync(job);
        
        // Publica ExecutionFinished
        await _publisher.PublishAsync(new ExecutionFinished { ... });
    }
}
```

Se ExecutionService NÃO tem isso implementado, é aí que está o problema!
