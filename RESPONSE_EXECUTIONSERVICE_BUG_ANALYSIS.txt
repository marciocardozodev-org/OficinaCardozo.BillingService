=================================================================================
RESPOSTA BILLINGSERVICE - Bug de SNS Subscription Confirmado
=================================================================================

EXECUÇÃO
========

Vocês estão **100% CORRETOS**. Identificamos a raiz do problema:

BillingService **está deletando PaymentConfirmed** que nunca deveria processar.


ANÁLISE DO BUG
==============

No arquivo SqsEventConsumerImpl.cs:

```csharp
public async Task ConsumeAsync(string eventType, string payload, ...)
{
    if (eventType == nameof(OsCreated))  
    {
        // ✅ Processa OsCreated (evento esperado)
        await _osCreatedHandler.HandleAsync(envelope);
    }
    // ❌ SE eventType == "PaymentConfirmed":
    //    - NÃO entra no if
    //    - NÃO processa nada
    //    - MAS a mensagem é deletada mesmo assim!
}
```

**O fluxo errado que estava acontecendo:**

1. OsService publica OsCreated
   ↓
2. SNS os-created → SQS billing-events
   ↓
3. BillingService consome OsCreated ✅
   └─ Cria orçamento
   └─ Publica PaymentConfirmed em SNS payment-confirmed
   ↓
4. SNS payment-confirmed → SQS billing-events (SUBSCRIPTION ERRADA!)
   ↓
5. BillingService consome de billing-events
   ├─ Recebe PaymentConfirmed
   ├─ Checa: "é OsCreated?" → NÃO
   ├─ Ignora completamente
   └─ **DELETA a mensagem!** ❌
   ↓
6. ExecutionService esperando, mas mensagem já foi deletada
   └─ Resultado: OS 5, 6, 7 bloqueadas


RESPONSABILIDADE DE BILLINGSERVICE
===================================

BillingService DEVE fazer:
  ✅ Consumir OsCreated de billing-events
  ✅ Processar com MercadoPago
  ✅ Publicar PaymentConfirmed em SNS payment-confirmed
  
BillingService NÃO DEVE fazer:
  ❌ Consumir PaymentConfirmed que ele mesmo publica
  ❌ Rotar PaymentConfirmed de volta para billing-events


SOLUÇÃO
=======

Há 2 opções. Recomendamos a OPÇÃO 1:

OPÇÃO 1: Remover a Subscription Errada (PREFERIDO)
---------------------------------------------------

A subscription de `payment-confirmed` → `billing-events` NUNCA deveria existir!

Remover:
```bash
aws sns unsubscribe --region sa-east-1 \
  --subscription-arn "arn:aws:sns:sa-east-1:953082827427:payment-confirmed:8058c497-e67a-4a68-89ab-cf76dc6f1fb0"
```

Status atual (ERRADO):
  ├─ payment-confirmed → billing-events ❌
  └─ payment-confirmed → execution-events ✅

Status desejado:
  └─ payment-confirmed → execution-events ✅ (APENAS!)

❌ NÃO deve haver: payment-confirmed → billing-events


OPÇÃO 2: Desabilitar SQS Consumer (Workaround, não definitivo)
--------------------------------------------------------------

Se não conseguir remover a subscription por limitações, poderíamos:
- Desabilitar SqsEventConsumerHostedService em BillingService
- Receber PaymentConfirmed apenas via MercadoPago webhook (já funciona)

MAS isso não é ideal porque deixa a infraestrutura confusa.
